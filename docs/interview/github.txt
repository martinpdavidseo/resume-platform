A workflow is a GitHub Actions pipeline defined in YAML that automates tasks such as validation, build, testing, deployment, and infrastructure provisioning.

Secrets are encrypted values stored in GitHub and used by workflows to securely access credentials, tokens, and sensitive information.

The collection of all these workflows is what we call GitHub Actions.
GitHub Actions = The Service

Workflow = Individual Pipeline
Runner = Machine executing the workflow
